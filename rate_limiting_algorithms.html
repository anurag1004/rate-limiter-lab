Token Bucket — Each request consumes 1 token. Tokens refill at a fixed rate up to capacity. Bursts are allowed as long as tokens remain. When empty, requests are rejected until refilled. Memory O(1).

Leaky Bucket — Requests enter a fixed-size FIFO queue. The queue drains at a constant rate regardless of burst. When full, new arrivals overflow (dropped). Output is always smooth — no downstream burst possible.

Sliding Window Log — Stores every request timestamp. On each request, prunes expired entries then checks count. Perfectly accurate, no boundary burst. Memory cost O(n) — expensive at high traffic.
